# Ampersend Marketplace Registration — Syra

Syra x402 APIs are **not yet listed** on [app.ampersend.ai/discover](https://app.ampersend.ai/discover). Production marketplace currently surfaces **catalog** (hand-curated) and **ampersend** (first-party) sources only — the `bazaar` source has zero listings as of 2026-07-02.

## What was wrong

1. **No Base settlement had indexed Syra in Bazaar** — Ampersend ingests Bazaar after at least one successful Base mainnet x402 payment.
2. **402 responses used `http://api.syraa.fun/...`** instead of `https://` — reverse-proxy URL mismatch blocked proper discovery metadata.
3. **402 `resource` lacked Bazaar service fields** (`serviceName`, `tags`, `iconUrl`) required for facilitator search indexing.
4. **Catalog curation** — most marketplace entries (Exa, BlockRun, etc.) are hand-added to `source=catalog`; email is still the reliable path.

## Fix (code + ops)

```bash
# Install ampersend skill + CLI (once per machine)
npx skills add edgeandnode/ampersend-sdk#skills/latest -a cursor -y
npm install -g @ampersend_ai/ampersend-sdk@latest

# After deploying API fixes:
cd api
npm run register-ampersend -- --pay
```

This validates 402 metadata, runs paid Base E2E on key endpoints, and writes `ampersend/catalog-registration.json`.
